Aprendiendo a usar GitHub

GitHub es una plataforma de desarrollo colaborativo donde los programadores y desarrolladores pueden almacenar, gestionar y compartir sus proyectos de código.

+ Información

como usar github

GitHub es una plataforma que integra diversas herramientas para facilitar su uso adecuado. Para dominarla, es fundamental comprender claramente los diferentes conceptos asociados a su funcionamiento.

Repositorio

Un repositorio es un espacio de almacenamiento para un proyecto, donde se guarda todo el código, archivos y el historial de cambios, es posible realizarlo de manera local o como normalmente se usa en GitHub.

Commit

Un commit es como un "punto de control" que guarda los cambios realizados en el proyecto en un momento específico. Estos contiene una descripción de los cambios, el autor, la fecha y un identificador único llamado hash.

Bases de Git

Git es un sistema de control de versiones distribuido. Esto significa que cada desarrollador tiene una copia completa del historial del proyecto en su computadora

Crear Una Rama

Una rama en Git es una versión separada del proyecto en la que es posible trabajar de manera independiente del código principal.

+ Información

Fusionar Ramas

Fusionar ramas es el proceso de combinar los cambios de una rama con otra. Normalmente, se fusionan las ramas secundarias o de trabajo con la rama principal cuando los cambios han sido probados y aprobados.

+ Información

Clonar Repositorio

Clonar un repositorio significa crear una copia exacta de un proyecto, almacenándola en nuestro dispositivo de manera local, esta descarga incluye el historial de cambios, ramas y todos los archivos que pueda contener.
